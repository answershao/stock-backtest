# 基于原文可直接提取的回测版规则

## 说明

本文件只整理 `html1/` 中作者明确写出的、可以直接量化的规则。

约束如下：

- 不补充本文档作者之外的推断
- 不为了“更完整”而擅自增加条件
- 不把不同阶段、不同体系的规则强行混合

因此，这份文档不是“我认为更合理的回测规则”，而是：

> 尽可能忠于原文表述的、可量化的规则摘录版。

## 一、主体系来源

当前可直接量化的主体系，最明确的来源是：

- `2018-11-20 极简估值教程——完结篇 价值投资系统建构举隅`

其他文章主要用于补充执行限制，不改动主参数。

## 二、主体系名称

按原文表述，这套体系可记为：

> 基于预估收益率的价值投资系统

## 三、变量

原文明确列出的变量如下：

1. `g`
2. `PEG`
3. `目标预估收益率`
4. `卖出预估收益率`
5. `组合中股票数量`
6. `行业数量`
7. `行业最大股票数量`
8. `n`
9. `g可信值`

## 四、决策公式

### 1. PEG

原文公式：

`PEG = PE / (g * 100)`

说明：

- 原文中的 `g` 以小数形式使用时，在 PEG 公式中乘以 `100`

### 2. 预估收益率

原文使用的核心估算逻辑为：

`预估年化收益率 = ((未来PE / 当前PE)^(1/n)) * (1 + g) - 1`

原文默认：

- `未来PE` 采用历史均值附近
- `n = 3`
- `g` 采用券商未来三年增长预估

### 3. g 可信值处理

原文明确写出：

- `g可信值 <= 35%`

即：

- 如果给出的未来三年复合增长预期高于 `35%`
- 则一律按 `35%` 处理

## 五、原文给出的参数

以下参数来自原文示例，原文同时说明“参数不一定合理，可按个人情况自由选取”。

但若要求忠于原文进行回测，则应优先采用以下默认值：

1. `g > 10%`
2. `PEG < 1.2`
3. `目标预估收益率 > 35%`
4. `卖出预估收益率 <= 0%`
5. `组合中股票数量 = 20`
6. `行业数量 = 10`
7. `行业最大股票数量 = 5`
8. `n = 3`
9. `g可信值 <= 35%`

## 五点五、核对后的补充说明

继续核对 `2018-10-28`、`2018-11-02`、`2018-11-10` 等估值教程前序文章后，需要补上几条原文已经明确说过、但不应误写成“唯一标准”的说明。

### 1. `PEG = 1.0` 是早期“合理估值”参考值

在 `2018-10-28 极简估值教程——第一讲 速判估值与PEG的推导` 中，作者明确写出：

- `PEG = 1.0` 为合理估值参考
- `0.8 ~ 1.2` 可视作合理区间
- 明显大于 `1.0` 可视作高估
- 明显小于 `1.0` 可视作低估

因此：

- 后来 `2018-11-20` 文中的 `PEG < 1.2`，并不是凭空出现的阈值
- 它和前文的“`0.8 ~ 1.2` 为合理区间”是一致的

### 2. `g > 10%` 是系统参数，不等于作者只认可 `10%` 以上公司

在 `2018-10-28` 中，作者还写过：

- `一般 g 取 15% ~ 25% 比较恰当`

这句话是对常见估值取值习惯的说明，不是后来系统文中的硬性买入条件。

因此忠于原文时应这样处理：

- `g > 10%` 仍然是 `2018-11-20` 主体系里的明确筛选条件
- `15% ~ 25%` 只作为早期估值讨论中的经验描述，不应强行改写主体系阈值

## 六、买入条件

原文明确写出的买入条件如下：

1. `PEG < 1.2`
2. `目标预估收益率 > 35%`
3. `当前市盈率必须小于等于历史均值`
4. `g > 10%`
5. `组合中股票数量不大于 20`
6. `未超过行业最大股票数量 5`

### 买入条件的原文背景补丁

在 `2018-11-02 历史估值的参考与运用` 与 `2018-11-10 安全边际` 中，作者明确强调了两件事：

1. 历史估值均值具有很强参考意义
2. 低估买入的核心目的是获得安全边际

因此，`当前市盈率必须小于等于历史均值` 这一条，不只是机械筛选，而是原体系里“安全边际”的直接落地形式。

## 七、卖出条件

原文写明：

> AB 任意一个发生即卖出

### A. 基本面变坏

- `g < 10%`

### B. 个股处于极端高估或未来预估收益率降低

- `卖出预估收益率 <= 0%`

## 八、换股条件

原文明确写出的换股条件如下：

### 1. 换入与换出之间的收益差

- `换入标的与换出标的目标预估收益率之差 > 35%`

### 2. 换入标的本身必须满足

1. `PEG < 1.2`
2. `目标预估收益率 > 35%`
3. `当前市盈率必须小于等于历史均值`
4. `g > 10%`

### 3. 换股后组合仍需满足

1. `组合中股票数量不大于 20`
2. `未超过行业最大股票数量 5`

## 九、资金管理

以下内容原文写得非常明确：

### 1. 等权分配

- `资金平均分成 20 份`

原文补充：

- 你设定的组合股票数量是几就分几份

### 2. 行业分配

- `行业数量设置为 10`
- `每个行业最多 5 只`

### 3. 允许空仓

原文明确允许：

- 如果满足条件的股票不足 20 只，则剩余资金空仓

原文举例：

- 如果只选出 `8` 只，那就有 `12` 份资金空仓

### 4. 再平衡频率

原文写明：

- `定期半年或一年再平衡一次`

### 5. 再平衡方法

原文方法为：

1. 用总资产除以 `20` 份，求得当前每份资金量
2. 卖出占用资金高的个股
3. 买入占用资金低的个股
4. 使每份资金接近当前每份资金量

## 十、预估收益率示例

原文示例：

- 当前 `PE = 10`
- 历史均值 `PE = 20`
- 券商未来三年增长率 `g = 20%`

则：

`((20 / 10)^(1/3)) * (1 + 20%) - 1 = 51.19%`

原文据此判断：

- 预估收益率大于 `35%`
- 若同时 `PEG = 0.5`
- 则可以考虑买入

## 十一、原文关于卖出阈值的说明

原文对参数解释非常明确：

- 买入应“严进”
- 卖出应“宽出”

其直接量化体现就是：

- 买入要求：`目标预估收益率 > 35%`
- 卖出要求：`卖出预估收益率 <= 0%`

原文强调：

- 持有中如果未来预估收益还有 `10%`
- 也未必该卖

因此，在忠于原文的前提下，不应把卖出阈值改成更高值。

## 十三点五、继续核对后的“非冲突但不同层级”内容

继续核对后，发现以下内容与主体系并不冲突，但层级不同，不能直接替代主体系参数。

### 1. “最佳开仓点”和“最佳平仓点”属于原则，不属于新增数值规则

来源：

- `2019-01-06 基本面真谛`

- 最佳开仓点：`主逻辑坚实，行情受次逻辑干扰时`

这与主体系并不冲突，但它们不是可直接替代 `g / PEG / 预估收益率` 的数值条件。

因此：

- 可作为解释层原则保留
- 不应被我擅自翻译成新的数值阈值

### 2. “低估的不传之秘”不能直接并入统一回测规则

来源：

- `2018-11-02 历史估值的参考与运用`

原文明确写出一个经验判断：

> 找最近一次市场崩溃时该股票的估值，这个估值基本上就是该股绝佳的低估入场值

但作者同时附带了重要前提：

- `条件是该股基本面不能比当时差太多`

因此这条更像：

- 人工估值经验

而不是已经被写成可以统一程序化执行的硬规则。

### 3. 实盘展示过 `25` 只组合，但不构成对 `20` 只主体系的否定

来源：

- `2019-01-28 为何多只股票暴跌，组合仍然稳如泰山`

该文写到作者某次实盘/展示组合为：

- `25` 只股票

同时还给出了核心/卫星结构：

- 医疗 `8` 只
- 消费 `12` 只
- 金融 `4` 只
- 科技 `1` 只

这说明：

- 作者在实盘展示中确实用过 `25` 只组合

但这并不自动推翻 `2018-11-20` 那篇系统文的：

- `组合股票数量 = 20`

因为原文自己已经说过：

- 参数只是举例
- 可以按自己的情况选取

所以忠于原文时，最稳妥的写法是：

- `20` 只仍是主体系默认参数
- `25` 只说明作者实盘中存在参数变体

## 十三点六、进一步核对后的原文补丁

继续核对以下三篇后：

- `2019-03-06 价值投资的防守与离场`
- `2021-01-09 投资组合该如何应对类似2008年暴跌70%的行情呢？`
- `2022-10-19 要避免重大的操作误区：市场底与估值底该怎样协调`

没有发现会推翻主体系的数值冲突，但有三条需要明确写入的原文补丁。

### 1. 价值组合体系不以“价格止损”作为核心部件

在 `2019-03-06 价值投资的防守与离场` 中，作者明确写出：

- 投资体系有两种：`择时` 与 `组合`
- `止损` 是择时法的重要部件
- 而价值投资是典型的组合法

因此，忠于原文时应明确：

- 当前主体系不是“价格止损驱动系统”
- 不应自行给它补一个固定百分比止损

同文还补充了作者列举的主动离场规则：

1. `三年不盈利离场`
2. `一旦看错公司基本面，出现重大失误，无论涨跌，立即离场`
3. `公司出现高估离场`
4. `发现更好的投资标的，可以换入更优标的`

其中：

- 第 2、3、4 条与当前回测版主体系并不冲突
- 第 1 条 `三年不盈利离场` 是新出现的原文明示规则

但要注意，这条规则在 `2018-11-20` 的系统文中并未写成数值卖出条件，因此：

- 它应作为“原文存在的补充离场规则”保留
- 不应未经说明就覆盖主体系里 `g < 10%` / `预估收益率 <= 0%` 的卖出框架

### 2. 极端暴跌情景下，作者明确保留预备队资金

在 `2021-01-09 投资组合该如何应对类似2008年暴跌70%的行情呢？` 中，作者明确给出一条极端场景应对规则：

- `预备队占总资本 10% ~ 20%`

原文组合防守表达为：

- `好公司`
- `组合`
- `不加杠杆`
- `不重仓`
- `预备队(占总资本10～20%)`

这意味着：

- 作者在极端系统性暴跌场景下，明确承认保留 `10% ~ 20%` 现金/预备队是其原文做法

因此回测文档应修正为：

- 虽然 `2018-11-20` 主体系允许因标的不够而自然空仓
- 但作者在后续文章里还明确提出过：极端风险情景中可主动保留 `10% ~ 20%` 预备队

### 3. 市场底确认后，不再死等个股“完美估值底”

在 `2022-10-19 要避免重大的操作误区：市场底与估值底该怎样协调` 中，作者明确写出：

- `市场底时听市场的，估值底时听估值的`

以及：

- 如果市场底已经形成、甚至已经企稳回升
- 对于个股或板块的估值底，`不等`

原文明确说明：

- 如果市场仍持续下跌，当然要等、要审慎
- 但如果市场已经探底回升，就不该死等自己主观设定的更低估值

因此，忠于原文时应补充：

- 主体系中的“当前PE小于等于历史均值”依旧成立
- 但在市场底已经确认的语境下，作者明确反对继续机械等待“更完美”的个股估值底

同时，原文还补了一个限制：

- 市场底时不是所有股票都可以莽撞去买
- `过于高估的`
- `炒作过度的`
- `没有基本面支撑的`
- 依旧要谨慎

这个限制与当前回测版并不冲突。

## 十三点七、继续核对“调仓/回撤/执行颗粒度”后的原文补丁

继续核对以下三篇后：

- `2023-07-10 为什么投资组合中的调整股是金疙瘩？`
- `2024-02-01 你们最大的问题是没过“回撤关”`
- `2022-08-30 基金操盘手法极为粗糙的原因，个人投资者完全可以用细腻的手法超越基金`

仍未发现对主体系参数的硬冲突，但需要补上以下原文限制和实盘表达。

### 1. 高位减仓后，作者会保留 `10% ~ 20%` 观察仓

在 `2023-07-10 为什么投资组合中的调整股是金疙瘩？` 中，作者明确写出：

- 某些早年建仓的优质股，在高位时其仓位已减到最初的 `20%` 和 `10%`

原文语境明确表示：

- 大部分高位仓位已落袋
- 剩余的 `10% ~ 20%` 仓位属于继续持有的部分
- 作者还直接称之为 `观察仓`

因此，忠于原文时可以补充：

- 作者在实盘调仓中，存在对高估值老持仓减仓后仅保留 `10% ~ 20%` 仓位的表达

但也要注意：

- 这不是 `2018-11-20` 主体系里统一写出的硬参数
- 它属于后续文章中披露的实盘调仓做法

### 2. 组合中的“调整股”是再配置来源

同文还明确表达了一个重要的调仓边界：

- 从个股视角看，调整股很难拿
- 从组合视角看，调整股是“金疙瘩”

原因在于：

- 组合允许灵活加减仓
- 高位减下来的资金可以去加仓当时不被看好的低估优质股

因此，这与当前回测版并不冲突，但更精确地说明了作者的原文态度：

- 调整股不是单纯的“被套股”
- 它在组合语境里是下一轮再配置的资金来源与目标对象

### 3. 虚回撤与实回撤再次被明确区分

在 `2024-02-01 你们最大的问题是没过“回撤关”` 中，作者继续明确：

- `虚回撤`：已有浮盈后的持有回撤
- `实回撤`：卖出、止损导致的确定性损失

这与此前已补入的内容一致，没有冲突。

但这篇文章进一步强化了一个原文边界：

- 对于价值投资而言，不能因为回撤就切换成趋势止损思维
- 否则容易在两套体系之间反复横跳

因此，当前回测版关于“不能把趋势止损规则擅自并入价值组合主体系”的写法，已被再次印证。

### 4. 个人投资者的原文优势是“低位提前埋伏 + 慢慢建仓加仓”

在 `2022-08-30 基金操盘手法极为粗糙的原因，个人投资者完全可以用细腻的手法超越基金` 中，作者明确写出：

- 个人投资者可以利用资金小的优势
- `在估值低位提前埋伏，慢慢建仓、加仓`

并明确声称：

- 这样会大幅节约成本

这条并未推翻当前回测版的分批建仓表达，反而给它补上了更明确的原文来源。

因此可以保留为：

- 作者原文支持：低位提前埋伏、慢慢建仓加仓
- 但未写成更具体的分几笔、隔多久、每笔多少比例

所以不应被我扩写成更细的机械规则

## 十三点八、继续核对“净资产/长期持有/资产配置”后的原文补丁

继续核对以下三篇后：
