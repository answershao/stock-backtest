# 交易策略研究回测设计文档

## 1. 文档目的

本文档用于定义一套可实际落地的研究版回测策略。

该策略的来源是对既有交易思想的工程化重写，而不是对原文规则的逐字复现。本文档关注的是：

- 如何把策略思想转成可执行系统
- 如何定义可回溯的数据口径
- 如何让买入、卖出、减仓、换股和再平衡规则闭合
- 如何为后续代码实现提供统一设计基线

相关文档：

- [trading_strategy.md](/root/shaopf/stock-backtest/docs/trading_strategy.md)：手册版
- [trading_strategy_backtest.md](/root/shaopf/stock-backtest/docs/trading_strategy_backtest.md)：原文摘录版

## 2. 设计目标

本设计文档面向以下目标：

1. 尽量保留“低估买入、高估减卖、组合持有、低频调仓”的核心思想。
2. 尽量使用历史上可以真实获取的数据，避免未来函数。
3. 对无法直接取得的原始条件，定义明确的研究替代规则。
4. 使策略规则能够被程序完整执行，而不是停留在原则层。
5. 为后续回测代码、参数配置和结果解释提供统一口径。

## 3. 非目标

本文档不承担以下目标：

1. 不证明该策略优于其他策略。
2. 不宣称已精确复现作者实盘系统。
3. 不把所有主观投资判断强行机械化。
4. 不负责描述 UI、报表展示样式或部署方案。

## 4. 设计原则

系统设计遵守以下原则：

1. 原文规则与研究假设必须严格区分。
2. 模糊理念不伪装成精确规则。
3. 默认优先选择简单、稳定、可解释的量化替代方案。
4. 所有规则必须能在调仓时点被程序判断。
5. 参数应集中配置，避免散落在实现代码中。

## 5. 策略概述

本策略是一个研究版价值组合系统，具有以下特征：

- 价值导向
- 低频调仓
- 组合分散
- 等权为主
- 估值驱动的买卖决策

与原文体系相比，最大的工程化变化是：

- 不再依赖难以稳定获得的“未来预估收益率”
- 改用 `PE_TTM` 历史均值与历史分位来定义低估、高估和极端高估
- 改用历史利润增速或一致预期代理变量定义 `g`

## 6. 术语与定义

本文档统一使用以下术语：

- `PE_TTM`：滚动十二个月市盈率
- `PB`：市净率
- `ROE`：净资产收益率
- `g`：增长变量，优先表示未来 3 年增长预期；若无，则使用历史利润增速代理
- `PEG`：`PE / (g * 100)`
- `PE_MEAN`：滚动窗口内 `PE_TTM` 历史均值
- `PE_P80`：滚动窗口内 `PE_TTM` 历史 `80%` 分位数
- `PE_P95`：滚动窗口内 `PE_TTM` 历史 `95%` 分位数
- `标准仓位`：单票目标资金占组合总资产的比例，默认 `5%`
- `观察仓`：减仓后保留的小比例持仓，本文档统一量化为 `50%` 标准仓位

## 7. 系统边界

本系统的职责包括：

- 构建候选池
- 计算估值与质量因子
- 执行买入、卖出、减仓、换股与再平衡
- 生成净值、交易明细、持仓明细和绩效指标

本系统当前不负责：

- 自动识别“主逻辑破坏”这类主观事件
- 自动抽取作者文章中的非结构化观点
- 宏观择时信号的高频交易实现

## 8. 数据需求

### 8.1 价格数据

必需字段：

- `trade_date`
- `close_hfq`

用途：

- 净值计算
- 调仓成交
- 持仓市值计算

### 8.2 分红送转数据

必需字段：

- `ex_date`
- `cash_dividend_per_share`
- `bonus_ratio`
- `transfer_ratio`

用途：

- 更新现金
- 更新持仓股数

### 8.3 基本面数据

最小必需字段：

- `PE_TTM`
- `PB`
- `ROE`
- `industry`
- `net_profit_growth`

说明：

- `net_profit_growth` 是研究版增长变量的基础来源
- 若存在历史一致预期数据，可替代或补充该字段

### 8.4 交易日历

必需字段：

- 交易日集合

用途：

- 调仓日顺延
- 日频净值推进

## 9. 数据口径设计

### 9.1 增长变量 `g`

`g` 的取值按以下优先级确定：

1. 使用历史一致预期的未来 3 年复合增速
2. 若无一致预期，则使用最近 3 年归母净利润复合增速
3. 若仍不足，则降级为最近 2 年复合增速
4. 若仍无法计算，则使用最近 1 年利润增速

异常值处理：

- `g > 35%` 时按 `35%` 截断
- `g <= -100%` 时视为缺失

### 9.2 历史估值窗口

估值统计采用滚动窗口：

- 窗口长度：`5` 年
- 采样频率：调仓日
- 有效样本：仅统计 `PE_TTM > 0`

样本不足处理：

- 有效样本少于 `36` 个月时，不参与估值分位判断

### 9.3 行业分类

系统必须选定单一行业分类标准，并在全回测期内保持一致。

推荐做法：

- 使用数据源自带一级行业分类
- 历史分类缺失时按最新可用分类回填，并单独记录该假设

## 10. 候选池设计

### 10.1 候选池模式

系统支持以下候选池模式：

1. 全 A
2. 宽基成份股池
3. 人工白名单池

默认建议：

- 先用人工白名单池启动实现

理由：

- 更接近策略长期偏好
- 数据清洗成本更低
- 能先验证规则本身是否闭合

### 10.2 基础排除条件

调仓日若股票满足任一条件，则从候选池排除：

1. `PE_TTM <= 0`
2. 基本面字段缺失
3. 上市时间不足 `3` 年
4. 行业分类缺失
5. 最近一期出现明显亏损或利润大幅恶化

第 `5` 条若暂时无法标准化，可先保留为可选规则。

## 11. 因子设计

### 11.1 PEG

定义：

`PEG = PE_TTM / (g * 100)`

约束：

- `g` 必须使用小数格式
- 若 `g <= 0`，则该股票不参与 PEG 买入判断

### 11.2 历史均值估值

定义：

- `PE_MEAN = rolling_mean(PE_TTM, 5y)`

用途：

- 作为低估买入条件的一部分

### 11.3 历史分位估值

定义：

- `PE_P80 = rolling_quantile(PE_TTM, 0.8, 5y)`
- `PE_P95 = rolling_quantile(PE_TTM, 0.95, 5y)`

用途：

- `PE_P80` 用于减仓
- `PE_P95` 用于清仓

### 11.4 排序得分

当候选股票多于可持仓上限时，使用排序得分。

默认研究版公式：

`score = g * 100 - PEG * 10 - pe_percentile * 20 + ROE`

说明：

- `pe_percentile` 取值范围为 `0` 到 `1`
- 该公式仅为默认实现，不代表最终最优表达

## 12. 交易规则设计

### 12.1 买入规则

调仓日满足以下条件时，股票进入买入候选名单：

1. `g > 10%`
2. `PEG < 1.2`
3. `PE_TTM <= PE_MEAN`
4. `ROE >= 12%`
5. 历史估值样本满足要求

说明：

- `ROE >= 12%` 属于研究版质量过滤，不属于原文硬规则

### 12.2 卖出规则

满足任一条件时清仓：

1. `g < 10%`
2. `PE_TTM >= PE_P95`
3. 公司进入亏损或 `PE_TTM` 失效

### 12.3 减仓规则

满足任一条件时减仓至观察仓：

1. `PE_TTM >= PE_P80`
2. `PEG >= 1.8`

观察仓定义：

- 保留 `50%` 标准仓位

### 12.4 换股规则

当组合已满，且存在未持有股票满足全部买入条件时，允许触发换股。

触发条件：

- `new_score - old_score >= 20`

换出顺序：

1. 优先换出得分最低股票
2. 若得分接近，优先换出估值分位更高者

## 13. 组合与仓位设计

### 13.1 组合规模

默认约束：

- 最大持仓数：`20`
- 最大行业数：`10`
- 单行业最大持仓数：`5`

### 13.2 仓位设计

默认约束：

- 标准仓位：`5%`
- 单票上限：`10%`

### 13.3 现金设计

系统允许两种现金来源：

1. 标的不够时的被动空仓
2. 策略主动保留的预备队

默认实现：

- 不主动保留预备队
- 仅保留被动空仓

### 13.4 杠杆设计

默认约束：

- 禁止融资
- 禁止杠杆

## 14. 调仓与执行流程

### 14.1 调仓频率

默认：

- 每半年调仓一次

可选：

- 每年调仓一次

### 14.2 调仓日执行顺序

每个调仓日按以下顺序执行：

1. 更新因子与估值统计
2. 执行清仓规则
3. 执行减仓规则
4. 根据买入规则生成候选清单
5. 根据排序和行业约束补充持仓
6. 对现有持仓做等权再平衡

### 14.3 再平衡方法

再平衡目标：

- 使持仓尽量靠近标准仓位

执行方法：

1. 用当前总资产除以目标持仓数，得到目标资金
2. 卖出超配持仓
3. 买入低配持仓
4. 受最小交易单位和现金约束影响时，允许存在偏差

1. 日度净值序列

## 17. 研究假设分层

为避免误读，本文档中的规则分为三层：

### 17.1 原始思想保留层

保留的核心思想包括：

- 低估买入
- 高估减卖
- 组合分散
- 等权持有
- 低频调仓

### 17.2 研究替代层

替代规则包括：

- 用历史利润增速代理 `g`
- 用 `PE_TTM` 历史分位代理卖出预估收益率
- 用排序得分差代理换股收益差

### 17.3 工程实现层

工程参数包括：

- 窗口长度
- 调仓日历
- 样本缺失处理
- 行业分类来源
- 交易成本与成交规则

## 18. 风险与局限

本设计存在以下局限：

1. 不能完整复现主观“主逻辑强弱”判断。
2. 不能准确复现人工对公司质地的理解。
3. `g` 的代理变量与真实一致预期并不等价。
4. `PE` 分位高估只是近似卖出规则。
5. 候选池质量会显著影响回测结果。

因此本设计适合：

- 研究策略轮廓
- 做参数对比
- 迭代可执行规则

不适合：

- 声称已经完全还原原始实盘体系

## 19. 实现建议

建议按以下顺序落地：

1. 固定候选池
2. 接入价格、分红、基本面和行业数据
3. 实现滚动估值统计
4. 实现买入、清仓、减仓和换股
5. 实现半年调仓与等权再平衡
6. 接入绩效指标与报表输出

## 20. 结论

本设计文档定义的是一套研究版价值组合回测系统。

它保留了原策略的核心方向，但将关键判断重写为可执行规则，尤其是：

- 用 `g` 代理变量处理增长判断
- 用 `PE_TTM` 历史均值和历史分位处理估值判断
- 用组合和行业约束处理风险分散

后续代码实现应以本文档为设计基线，在实现过程中若发生口径变更，应优先回写本文档，而不是让代码自行演化出隐含规则。
