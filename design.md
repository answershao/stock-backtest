## 2. 项目概览

本项目是一个基于 Tushare 数据的股票研究与回测项目，当前主要覆盖三条主流程：

1. 缓存 Tushare 数据到本地
2. 基于缓存绘制历史三年预估年化收益率时序
3. 基于三年预估年化收益率信号做组合回测

当前推荐直接使用根目录脚本：

```bash
python prefetch_cache.py ...
python plot_expected_return.py ...
python run_backtest.py ...
```

这样可以直接对应到实际业务入口，减少额外封装层。

### 2.1 项目结构

推荐优先关注以下命名空间：

1. 根目录脚本：`prefetch_cache.py`、`plot_expected_return.py`、`run_backtest.py`
2. `backtest.data`：数据源、缓存与 Tushare 适配
3. `backtest`：预估收益率计算与策略回测引擎
4. `backtest.stock_pool`：股票池解析与加载

## 3. 系统目标

本系统面向 A 股日频股票策略回测，目标如下：

1. 支持股票池内多股票组合回测。
2. 支持按固定调仓日执行低频调仓。
3. 支持按统一公式计算个股三年预估年化收益率。
4. 支持固定持仓数量、固定单票仓位的等权组合。
5. 支持明确的退出、补位和再平衡机制。
6. 支持输出净值、持仓、交易明细和绩效指标。

---

## 4. 策略定义

### 4.1 策略名称

可将本策略定义为：

`基于 PE-G 三年预估年化收益率的 Top20 等权轮动策略`

### 4.2 核心思想

在给定股票池内，对每只股票计算三年预估年化收益率，并按该指标排序。

组合管理规则为：

1. 回测起始日计算全部股票的 `expected_return_3y`，仅买入 `expected_return_3y > 20%` 的股票。
2. 最多持有 `20` 只，每只目标仓位固定为 `5%`。
3. 固定调仓日为每年 `05-01`、`11-01`，非交易日顺延。
4. 持仓股票若三年预估年化收益率小于 `0`，则退出。
5. 退出后，仅从未持仓且 `expected_return_3y > 20%` 的股票中按收益率从高到低补入。
6. 若合格标的不足 `20` 只，则剩余资金保留为现金。
7. 调仓日对最终持仓执行等权再平衡。

### 4.3 组合约束

- 最大持仓数：`20`
- 单票目标仓位：`5%`
- 总目标权益仓位上限：`100%`
- 若有效标的不足 `20` 只，允许保留现金

## 5. 三年预估年化收益率定义

### 5.1 公式

个股三年预估年化收益率定义为：

```python
expected_return_3y = ((target_pe / current_pe) ** (1 / 3)) * (1 + g) - 1
```

其中：

- `current_pe`：当前时点的 `PE_TTM`
- `target_pe`：未来估值锚，默认取过去十年 `PE` 参考中枢
- `g`：三年复合年化增长率，使用小数表示

### 5.2 变量解释

#### `current_pe`

- 使用调仓日可见的滚动市盈率 `PE_TTM`
- 若 `PE_TTM <= 0`，则该股票不参与排名和建仓

#### `target_pe`

默认定义为：

- 最近 `10` 年历史窗口内的 `PE_TTM` 参考中枢

默认计算方式：

- 仅统计 `PE_TTM > 0` 的有效样本
- 先使用最近 `10` 年 `PE_TTM` 的算术均值

异常值处理：

- 若 `PE` 历史序列存在明显异常值或极端值，不直接使用简单均值
- 应优先使用稳健口径替代，例如：
1. 截尾均值
2. 去极值后的均值
3. 中位数
4. 中位数附近区间均值

默认建议：

1. 先对最近 `10` 年正 `PE_TTM` 做去极值处理
2. 再计算处理后的均值作为 `target_pe`

若样本质量仍不足，则可退化为：

1. 最近 `10` 年 `PE_TTM` 中位数
2. 最近 `5` 年稳健均值

若历史样本仍不足最低要求，则该股票不参与排名

#### `g`

定义为三年复合年化增长率，取值优先级如下：

1. 未来三年一致预期复合增速
2. 最近三年净利润复合增速
3. 最近两年净利润复合增速
4. 最近一年净利润增速

异常处理：

- `g > 35%` 时按 `35%` 截断
- `g <= -100%` 时视为无效

### 5.3 无效股票判定

若股票满足以下任一条件，则该调仓日不参与策略排名：

1. `current_pe` 缺失
2. `current_pe <= 0`
3. `target_pe` 缺失
4. `g` 缺失
5. 无法计算 `expected_return_3y`

## 6. 股票池定义

### 6.1 股票池来源

本系统的股票池由用户自行定义，当前默认规模为 `27` 只。

股票池定义方式为：

1. 用户手工指定股票列表
2. 每只股票以股票代码为唯一标识
3. 可选附带股票名称、行业等辅助信息

系统不以全市场自动选池或指数成份自动选池为默认模式。

### 6.2 基础过滤条件

股票池中的股票在调仓日需先通过基础过滤，建议包括：

1. 上市时间不少于 `3` 年
2. 非 ST
3. 行业分类存在
4. 基本面字段完整
5. 当日存在可用价格数据

### 6.3 筛选规则

在用户自定义股票池基础上，策略应进一步执行筛选，筛选分为四层：

1. 优先覆盖的方向
2. 能力圈筛选
3. 基本面筛选
4. 估值筛选

#### 6.3.1 优先覆盖的方向

优先覆盖以下方向：

1. 消费
2. 医药
3. 家电
4. 金融
5. 部分制造业龙头
6. 与内需、民生、日常刚需相关的行业

进一步偏好：

1. 产品和需求逻辑容易理解的公司
2. 利润来源清晰的公司
3. 具备稳定需求和长期可持续性的行业
4. 大股东背景较硬、可理解性较强的公司

#### 6.3.2 尽量回避的方向

尽量回避以下对象：

1. 纯概念炒作
2. 高波动热点轮动
3. 难以理解的高科技故事股
4. 信息不透明、黑箱属性强的公司
5. 高度依赖短期博弈的题材股

对普通投资者尤其应谨慎的对象：

1. 高估值科技股
2. 依赖高位放量板接力的玩法
3. 以热点轮动为核心的操作习惯
4. 单纯依赖股息、忽视本金风险的死守型银行股策略

#### 6.3.3 能力圈筛选

只做自己看得懂的行业和公司：

1. 长期需求存在
2. 商业模式能理解
3. 需求逻辑能理解
4. 利润来源能理解

如果需要持续高频跟踪、持续解释、持续猜测，通常不适合进入候选池。

#### 6.3.4 基本面筛选

优先保留满足以下特征的公司：

1. 多年 `ROE` 较高
2. `ROE` 优选 `15%` 以上
3. 负债结构相对干净
4. 现金流较稳定
5. 分红能力较好
6. 利润留存后能够持续推动净资产增长

排除以下对象：

1. 逻辑不清
2. 增长不可持续
3. 财务质量差
4. 争议过大且难以判断真伪
5. 自己无法解释其价值来源

#### 6.3.5 估值筛选

估值层面的硬条件如下：

1. `g > 10%`
2. `PEG < 1.2`
3. `current_pe <= target_pe`

补充参考：

1. `PEG = 1.0` 可视为合理估值的中心参考
2. `0.8 ~ 1.2` 可视为大致合理区间
3. 优先保留处于历史低估区、低估附近或合理偏低区间的股票
4. 必要时可同时参考 `PB` 及其历史区间

筛选通过后，剩余股票才进入收益率排序环节。

## 7. 建仓规则

### 7.1 初始建仓时点

在回测起始日执行初始建仓。

### 7.2 建仓步骤

1. 计算股票池内全部有效股票的 `expected_return_3y`
2. 筛选出 `expected_return_3y > 20%` 的股票
3. 按该指标从高到低排序，最多选择前 `20` 只股票作为目标持仓
4. 对目标持仓等权分配，每只目标仓位 `5%`
5. 根据资金、交易单位和成本完成实际买入

### 7.3 现金处理

若有效股票少于 `20` 只，则：

- 仅买入有效股票
- 未分配资金保持现金

---

## 8. 调仓规则

### 8.1 调仓频率

系统默认每年两个固定调仓日：

- `05-01`
- `11-01`

### 8.2 调仓日顺延规则

若预设调仓日不是交易日，则顺延至下一个交易日。

### 8.3 调仓主流程

每个调仓日执行以下步骤：

1. 更新当日可用因子和估值数据
2. 评估当前持仓退出条件
3. 清仓应退出股票
4. 在未持仓且 `expected_return_3y > 20%` 的股票中寻找补位标的
5. 补足至目标持仓数或候选耗尽
6. 对最终组合执行等权再平衡

---

## 9. 退出规则

### 9.1 退出条件

持仓股票在调仓日若满足以下条件，则退出：

- `expected_return_3y < 0`

### 9.2 退出动作

对满足退出条件的股票执行清仓卖出。

### 9.3 退出后的状态

退出后释放的资金进入现金账户，等待补位或保留为现金。

---

## 10. 补位规则

### 10.1 补位触发

若清仓后持仓数量小于 `20`，则触发补位。

### 10.2 补位范围

补位范围限定为：

- 当前股票池中
- 当前未持仓
- 当前 `expected_return_3y > 20%`
- 当前调仓日可计算 `expected_return_3y`

的股票。

### 10.3 补位顺序

按 `expected_return_3y` 从高到低排序，依次补入。

### 10.4 补位停止条件

满足以下任一条件时停止补位：

1. 持仓数量达到 `20`
2. 没有可补位标的
3. 资金不足以形成最小可交易仓位

---

## 11. 再平衡规则

### 11.1 再平衡目标

所有最终入选持仓均按统一目标仓位配置：

- 单票目标仓位 `5%`

### 11.2 再平衡执行逻辑

对于最终持仓池中的每只股票：

1. 计算当前持仓市值
2. 计算目标持仓市值
3. 若当前持仓高于目标，则卖出超出部分
4. 若当前持仓低于目标，则买入不足部分

### 11.3 仓位偏差容忍

由于交易单位、手续费和现金限制，允许实际仓位与目标仓位存在小幅偏差。

可配置：

- 仓位容忍度，例如 `0% ~ 1%`

---

## 12. 交易执行规则

### 12.1 成交价格

日频回测默认支持以下成交价模式：

1. 调仓日收盘价成交
2. 下一交易日开盘价成交

系统实现时应固定选择一种，且全回测保持一致。

### 12.2 最小交易单位

A 股股票默认最小买卖单位为：

- `100` 股

买卖股数需向下取整到 `100` 股整数倍。

### 12.3 交易成本

系统应支持以下成本项：

1. 佣金
2. 印花税
3. 过户费

### 12.4 现金约束

买入时必须满足：

- 现金足以覆盖成交金额和全部交易成本

若现金不足，则按可承受最大股数缩减订单。

### 12.5 不可交易情形

以下情况默认不可成交：

1. 当日无有效价格
2. 股票停牌
3. 价格异常缺失

如需更严格模拟，可扩展：

1. 涨停买不到
2. 跌停卖不出
3. T+1 卖出限制

---

## 13. 公司行为处理

### 13.1 分红

若持仓股票发生现金分红，则：

- 按持股数量增加现金

### 13.2 送股和转增

若持仓股票发生送股或转增，则：

- 按持股数量调整持仓股数

### 13.3 处理时点

公司行为在除权除息生效日处理，并在当日调仓前完成账户更新。

---

## 14. 账户模型

### 14.1 账户组成

账户由以下部分构成：

1. 现金
2. 股票持仓
3. 交易记录
4. 每日资产记录

### 14.2 每日总资产

每日总资产定义为：

```python
total_value = cash + sum(position_shares * close_price)
```

### 14.3 持仓状态

每只持仓至少记录：

1. 股票代码
2. 持股数量
3. 最近成交价
4. 最近市值

---

## 15. 回测流程

### 15.1 初始化

回测开始前完成以下准备：

1. 读取配置
2. 加载股票池
3. 加载行情、基本面、分红送转和交易日历
4. 初始化账户现金
5. 生成调仓日列表

### 15.2 日频推进流程

对每个交易日执行：

1. 处理公司行为
2. 判断是否为调仓日
3. 若为调仓日，则执行策略调仓
4. 按当日价格计算持仓市值
5. 记录日终总资产、现金和持仓

### 15.3 调仓日内部流程

调仓日内部顺序固定为：

1. 计算股票池因子
2. 评估当前持仓退出条件
3. 卖出退出股票
4. 评估未持仓股票并补位
5. 对最终持仓执行等权再平衡
6. 记录调仓日志和成交记录

6. 调仓前权重
7. 调仓后目标权重

---

## 19. 绩效指标

系统建议输出以下绩效指标：

1. 累计收益率
2. 年化收益率
3. 年化波动率
4. 最大回撤
5. 夏普比率
6. 卡玛比率
7. 超额收益
8. 年化换手率
9. 总交易成本
10. 总交易笔数
11. 调仓次数

---

## 20. 默认行为约定

若未特别说明，系统默认采用以下约定：

1. 调仓日若非交易日，则顺延到下一个交易日
2. 公司行为先于当日调仓执行
3. 单票目标仓位固定为 `5%`
4. 组合最多持有 `20` 只股票
5. `expected_return_3y < 0` 即触发清仓
6. 补位只在当前未持仓股票中进行
7. 候选不足时允许持有现金
8. 买卖股数按 `100` 股整数倍向下取整

---

## 21. 非目标

本版本系统不以以下能力为目标：

1. 分钟级或更高频回测
2. 多资产类别统一回测
3. 杠杆与融资融券
4. 衍生品交易
5. 复杂订单簿撮合
6. 主观宏观择时信号自动化

---

## 22. 总结

本系统是一套面向 A 股日频场景的低频组合回测设计。

其核心闭环非常明确：

1. 在股票池中计算三年预估年化收益率
2. 选取排名靠前股票进入组合
3. 在固定调仓日检查退出条件
4. 用未持仓高收益股票补位
5. 对最终组合执行固定 `5%` 等权再平衡

该设计的重点不是复杂择时，而是：

- 统一收益率口径
- 明确组合更新规则
- 保持交易执行和结果输出可复现
