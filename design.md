# 股票交易策略回测系统 - 设计文档

## 一、项目概述

本项目用于回测一个 A 股市场的定期调仓交易策略，评估策略在历史数据上的表现。

## 二、策略规则

### 2.1 基本参数

| 参数 | 数值 |
|------|------|
| 初始本金 | 500 万元 |
| 回测起始日期 | `BacktestConfig.start_date`（默认配置为 2021-01-04） |
| 回测结束日期 | `BacktestConfig.end_date`（默认配置为 2026-06-30） |
| 持仓数量 | 20 只股票 |
| 单只仓位 | 5%（等权配置） |
| 调仓日期 | `BacktestConfig.rebalance_schedule`（默认配置为 `["05-01", "11-01"]`） |
| 仓位误差容忍 | ≤ 0.1%（即单只仓位在 4.9% ~ 5.1% 之间） |
| 分红处理方式 | `BacktestConfig.dividend_mode`（默认配置为 `reinvest`） | `reinvest`=分红再投资，`cash`=分红留存在现金账户 |

### 2.2 调仓逻辑

1. 在调仓日（若为节假日则顺延至下一个交易日），以当日**收盘价**执行调仓。
2. 调仓并非更换股票，而是对现有持仓进行**权重复平衡**：卖出超出目标仓位的部分，买入不足目标仓位的部分。
3. 若某只股票在调仓日因停牌/涨跌停无法交易，则跳过该股票，待下一调仓日再处理。
4. 调仓后，每只股票市值占组合总市值的比例恢复为 5%，误差不超过 0.1%。
5. 非调仓日不做任何操作（买入并持有）。

### 2.4 初始建仓

- 回测从 `BacktestConfig.start_date` 开始建仓；若该日不是交易日，则顺延至下一个交易日。
- 初始建仓日以当日收盘价一次性买入股票池中的股票，目标为每只仓位 5%。
- 受 A 股 `100` 股整数手约束、停牌或现金不足影响，个别股票可能无法在首日买满目标仓位。
- 之后每个调仓日进行权重复平衡。

## 三、交易成本

| 费用项 | 费率 |
|--------|------|
| 佣金 | 万分之一（0.01%），免最低 5 元 |
| 印花税 | 卖出时 0.05%（万分之五） |
| 过户费 | 0.001%（十万分之一） |

- 交易成本在每次调仓时扣除。
- 买入时仅计算佣金 + 过户费，卖出时计算佣金 + 印花税 + 过户费。

## 四、回测设置

### 4.1 时间范围

回测时间范围由 `BacktestConfig` 控制，可在配置对象中修改：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `start_date` | `"2021-01-04"` | 回测起始日期；若非交易日则顺延到下一个交易日执行初始建仓 |
| `end_date` | `"2026-06-30"` | 回测结束日期 |

**调仓日自动生成规则：**

1. 将 `rebalance_schedule` 中每个 `MM-DD` 展开为该年度内的具体日期。
2. 合并所有年份的日期，按时间排序，仅保留 `start_date` 之后且不超过 `end_date` 的理论调仓日。
3. 每个理论调仓日，根据交易日历顺延至最近的下一个交易日，得到实际调仓日。

> 示例：默认参数下共生成 **11 个调仓日**（2021-05-01 ~ 2026-05-01），跨度约 5.5 年。

### 4.2 数据来源

选用 **Tushare**（需配置 Token），通过以下接口获取数据：

| 数据项 | Tushare 接口 | 说明 |
|--------|-------------|------|
| 日线行情（可配置复权） | `pro_bar()` | 返回开高低收、成交量、成交额等，默认不复权；可通过 `BacktestConfig.price_adj` 切换为 `qfq` / `hfq` |
| 历史分红送配 | `pro.dividend()` | 返回除权除息日、派息、送股、转增等字段，仅保留实施完成记录 |
| 交易日历 | `pro.trade_cal()` | 判断节假日、生成调仓日对应的实际交易日 |
| 基准指数行情 | `pro.index_daily()` | 获取沪深 300 等基准指数的日线数据 |

> 使用前需在运行配置中提供 `tushare_token`，并确保本地已安装 `tushare` 依赖。

### 4.3 特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 调仓日停牌 | 该股票不交易，顺延至复牌后首个交易日以收盘价处理 |
| 调仓日涨跌停 | 无法买入/卖出，顺延至可以成交的交易日 |
| 股票退市 | 在退市前最后一个交易日以收盘价强制卖出；若无法卖出则计入亏损 |

---

### 4.4 分红与复权处理

采用 **可配置复权价格 + 分红 API + 现金账户 + 可配置分红处理模式** 方案，其中默认使用**不复权价格**。

**核心思路：**

- 默认使用不复权价格，尽量贴近当时真实可成交价格。
- 若需要平滑展示价格走势，可切换为前复权或后复权。
- 分红事件通过 `pro.dividend()` 直接获取公告精确值，**不做复权因子反推**。
- 现金分红记入组合的**现金账户**，送股/转增直接调整持股数。
- 通过 `dividend_mode` 控制现金分红的后续处理：
  - `reinvest`：每个调仓日，现金账户中的分红余额参与再投资。
  - `cash`：分红长期留存在现金账户，不主动用于买入股票。

**数据准备（一次性）：**

```python
pro.dividend(ts_code="000538.SZ")
```

返回字段：`除权除息日`、`派息`（每 10 股派多少元）、`送股`（每 10 股送多少）、`转增`（每 10 股转多少）。
过滤 `进度 == "实施"` 的记录，转换为每股数值后按除权日建立事件表。

**组合日频模拟循环：**

```
每日循环：
  1. 市值计算：
     组合总市值 = Σ(持股数 × 当日收盘价) + 现金账户余额

  2. 分红事件（若当日为某只股票的除权日）：
     - 现金分红 → 现金账户 += 持股数 × 每股派息
     - 送股      → 持股数 *= (1 + 送股比例)
     - 转增      → 持股数 *= (1 + 转增比例)

  3. 调仓事件（若当日为调仓日）：
     a. 卖出超配：仓位占比 > 5.1% 的股票，卖出超出部分
     b. 买入低配：
        - `reinvest`：用卖出得到的现金 + 现金账户余额，买入仓位 < 4.9% 的股票
        - `cash`：仅使用卖出得到的现金买入，分红现金继续保留在现金账户
     c. 调仓后现金处理：
        - `reinvest`：若资金已全部用于补仓，则现金账户余额归零；若因停牌/涨跌停等未能完成买入，剩余现金继续留存
        - `cash`：现金账户余额持续累积，不清零
```

**示例：** 持有 10,000 股云南白药，API 返回某除权日每股派息 3 元、送股 0.2（10送2）：
- 现金账户 += 10,000 × 3 = 30,000 元
- 持股数 → 10,000 × 1.2 = 12,000 股
- 若 `dividend_mode = "reinvest"`，最近调仓日 30,000 元与其他资金一并用于再投资
- 若 `dividend_mode = "cash"`，30,000 元继续保留在现金账户

## 五、评价指标

回测完成后，输出以下核心指标：

| 类别 | 指标 | 说明 |
|------|------|------|
| 收益 | 累计收益率 | 期末净值 / 期初净值 - 1 |
| 收益 | 年化收益率 | (期末净值/期初净值)^(1/年数) - 1 |
| 收益 | 超额收益 | 相对基准指数的超额收益率 |
| 风险 | 年化波动率 | 日收益率标准差 × sqrt(252) |
| 风险 | 最大回撤 | 净值曲线上任意高点到后续低点的最大跌幅 |
| 风险调整 | 夏普比率 | (年化收益 - 无风险利率) / 年化波动率 |
| 风险调整 | 卡玛比率 | 年化收益 / 最大回撤 |
| 交易 | 年化换手率 | 平均每年调仓产生的交易额 / 平均净值 |
| 交易 | 总交易成本 | 累计佣金 + 印花税 + 过户费 |
| 对比 | 基准收益 | 同期基准指数的收益表现 |

### 运行说明

1. 安装依赖：`pip install -r requirements.txt`
2. 确保 `tushare` 可用，且运行配置中填写了有效的 `tushare_token`
3. 执行：`python3 main.py`

> 当前代码已完成一轮结构性重构：模块之间通过 `BacktestConfig` 显式传递配置，不再依赖入口脚本回写全局配置。
